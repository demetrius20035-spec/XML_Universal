# Универсальная выгрузка/загрузка данных XML для 1С

Модули обработки универсальной выгрузки/загрузки данных XML из/в 1С:Предприятие 8.3:

- `МодульОбъекта.bsl` — модуль объекта обработки (вся логика выгрузки/загрузки);
- `МодульФормы.bsl` — модуль управляемой формы.

Начиная с версии **2.2.0** в обработке появились механизм настроек и программный
запуск, начиная с **2.3.0** — выгрузка в несколько файлов для работы на больших базах
(см. раздел 3).

## 1. Механизм настроек (конфигураций) выгрузки

Расставленные на дереве метаданных флажки, отборы, период выгрузки и дополнительные
объекты теперь можно сохранить в XML-файл и в любой момент восстановить — не нужно
заново расставлять галочки после падения 1С или для повторной выгрузки через неделю.

Само дерево (`Объект.ДеревоМетаданных`) целиком не сериализуется — его строки содержат
несериализуемые значения (объекты метаданных, менеджеры, настройки построителя).
Вместо этого сохраняется *состояние* дерева:

- полные имена объектов метаданных (`Справочник.Номенклатура`, `Документ.…`)
  и значения флажков `Выгружать` / `ВыгружатьПриНеобходимости`;
- отборы построителя отчета (через `ХранилищеЗначения`);
- период выгрузки, дополнительные объекты для выгрузки и прочие параметры
  (`ВыгружатьСДокументомЕгоДвижения`, `ТипВыгрузкиПериодическихРегистров` и т.д.).

При загрузке настроек дерево строится заново (`Инициализация()`), после чего флажки
расставляются по сохраненным полным именам. Если какой-то объект метаданных в текущей
конфигурации отсутствует (конфигурация изменилась), настройка для него пропускается
с сообщением пользователю, остальные применяются.

### Экспортные методы модуля объекта

| Метод | Назначение |
|---|---|
| `ПолучитьНастройкиВыгрузки()` | Возвращает структуру настроек (можно сохранить куда угодно: файл, регистр сведений, хранилище настроек) |
| `УстановитьНастройкиВыгрузки(Настройки)` | Применяет структуру настроек к обработке |
| `СохранитьНастройкиВыгрузкиВФайл(ИмяФайла)` | Сохраняет настройки в XML-файл |
| `ЗагрузитьНастройкиВыгрузкиИзФайла(ИмяФайла)` | Загружает и применяет настройки из XML-файла |

### Кнопки на форме

В модуле формы добавлены обработчики команд:

- `СохранитьНастройкиВыгрузки` — собирает флажки с формы и предлагает сохранить
  файл `НастройкиВыгрузки.xml`;
- `ЗагрузитьНастройкиВыгрузки` — запрашивает файл настроек и восстанавливает
  флажки на дереве, период и дополнительные объекты.

Чтобы кнопки появились на форме, в конфигураторе нужно добавить на форму две команды
с именами `СохранитьНастройкиВыгрузки` и `ЗагрузитьНастройкиВыгрузки`, указать им
соответствующие действия и вывести кнопки в командную панель (например, на страницу
«Выгрузка»). Работает и в файловом, и в клиент-серверном варианте, включая веб-клиент
(файл передается через временное хранилище).

## 2. Программный запуск выгрузки/загрузки

Обработка встроена в конфигурацию — выгрузку и загрузку можно запускать кодом,
без открытия формы.

### Выгрузка по ранее сохраненному файлу настроек

```bsl
Обработка = Обработки.ВыгрузкаЗагрузкаДанныхXML.Создать(); // имя обработки в вашей конфигурации
Обработка.ВыполнитьВыгрузкуПоФайлуНастроек("C:\Настройки\НастройкиВыгрузки.xml", "C:\Данные\Выгрузка.xml");
```

То есть сценарий «расставил галочки — сохранил — через неделю повторил выгрузку»
выполняется одной строкой.

### Выгрузка с расстановкой флажков из кода

```bsl
Обработка = Обработки.ВыгрузкаЗагрузкаДанныхXML.Создать();
Обработка.Инициализация();

Обработка.УстановитьПризнакВыгрузки("Справочник.Номенклатура");
Обработка.УстановитьПризнакВыгрузки("Документ.РеализацияТоваровУслуг");
// при необходимости — период:
Обработка.ДатаНачала    = НачалоГода(ТекущаяДатаСеанса());
Обработка.ДатаОкончания = КонецДня(ТекущаяДатаСеанса());

Обработка.ВыполнитьВыгрузку("C:\Данные\Выгрузка.xml");
```

### Программная загрузка

```bsl
Обработка = Обработки.ВыгрузкаЗагрузкаДанныхXML.Создать();
Обработка.Инициализация();
Обработка.ПродолжитьЗагрузкуВСлучаеВозникновенияОшибки = Истина; // по желанию
Обработка.ВыполнитьЗагрузку("C:\Данные\Выгрузка.xml");
```

### Настройки без файла (например, хранение в регистре сведений)

```bsl
// сохранение
Настройки = Обработка.ПолучитьНастройкиВыгрузки();
ХранилищеНастроек = Новый ХранилищеЗначения(Настройки);
// ... записать ХранилищеНастроек в регистр сведений / справочник ...

// восстановление
Обработка.УстановитьНастройкиВыгрузки(ХранилищеНастроек.Получить());
Обработка.ВыполнитьВыгрузку(ИмяФайла);
```

Для внешней обработки вместо `Обработки.<Имя>.Создать()` используйте
`ВнешниеОбработки.Создать(ПутьКФайлуОбработки)` (методы выполняются на сервере
или в толстом клиенте/внешнем соединении).

## 3. Выгрузка в несколько файлов (большие базы)

На больших базах однофайловая выгрузка падает с ошибкой
«Превышен максимальный расход памяти сервера за один вызов»: вся таблица
материализуется в памяти результатом запроса, а вся выгрузка выполняется одним
серверным вызовом. Для решения добавлены **два независимых режима**, которые можно
включать по отдельности или вместе.

### Режим «Выгружать в отдельные файлы по объектам метаданных»

Реквизит `ВыгружатьВОтдельныеФайлыПоМетаданным` (Булево). Каждый объект метаданных
выгружается в **отдельный файл отдельным вызовом сервера** — счетчик расхода памяти
кластера сбрасывается между файлами. Константы группируются в один файл,
дополнительные объекты для выгрузки — в свой:

```
00001.Константы.xml
00001.Справочник_Контрагенты.xml
00001.Документ_ПриходныйКассовыйОрдер.xml
00001.ДополнительныеОбъекты.xml
```

### Режим «Многотомная выгрузка»

Реквизиты `МноготомнаяВыгрузка` (Булево) и `КоличествоОбъектовВТоме` (Число,
по умолчанию 10000). Поток выгрузки разбивается на тома: каждые N объектов текущий
файл корректно закрывается (со своим блоком предопределенных) и открывается
следующий. При этом меняется способ чтения данных: вместо запроса всей таблицы
(`ВЫБРАТЬ _.*`) сначала выбираются только ссылки, а данные читаются и записываются
пачками по N ссылок — в памяти в каждый момент находится одна порция.

```
00001.001.xml
00001.002.xml
...
```

### Комбинации режимов

| По метаданным | Многотомная | Результат |
|---|---|---|
| ☐ | ☐ | `00001.xml` — как раньше, один файл |
| ☑ | ☐ | `00001.Справочник_Контрагенты.xml`, … |
| ☐ | ☑ | `00001.001.xml`, `00001.002.xml`, … |
| ☑ | ☑ | `00001.Справочник_Контрагенты.001.xml`, `00001.Справочник_Контрагенты.002.xml`, … |

**Каждый создаваемый файл — полноценная самостоятельная выгрузка**: загружается
обычной командой «Загрузить данные» (или `ВыполнитьЗагрузку()`), файлы можно
загружать по одному в любом порядке. Объекты, выгруженные «по ссылке», могут
повторяться в нескольких файлах — при загрузке они просто перезаписываются,
это безвредно.

### Что добавить в конфигураторе

Реквизиты обработки:

| Реквизит | Тип |
|---|---|
| `ВыгружатьВОтдельныеФайлыПоМетаданным` | Булево |
| `МноготомнаяВыгрузка` | Булево |
| `КоличествоОбъектовВТоме` | Число (10, 0, неотрицательное) |

Элементы формы (на странице «Выгрузка», привязка к `Объект.<Реквизит>`):
два флажка и поле ввода числа. Обработчики не требуются — вся логика уже в модулях.

### Программный запуск

```bsl
Обработка = Обработки.ВыгрузкаЗагрузкаДанныхXML.Создать();
Обработка.ЗагрузитьНастройкиВыгрузкиИзФайла("C:\Настройки\НастройкиВыгрузки.xml");

// многотомная выгрузка одним вызовом (порционное чтение + тома)
Обработка.МноготомнаяВыгрузка = Истина;
Обработка.КоличествоОбъектовВТоме = 20000;
Обработка.ВыполнитьВыгрузку("C:\Данные\00001.xml"); // создаст 00001.001.xml, 00001.002.xml...

// либо: файл на каждый объект метаданных одним вызовом (для фонового задания)
Обработка.ВыполнитьМногофайловуюВыгрузку("C:\Данные\00001.xml");
```

`ВыполнитьВыгрузку()` получила третий необязательный параметр
`ФильтрСоставаВыгрузки` — полное имя объекта метаданных («Справочник.Номенклатура»),
`"#Константы"` или `"#ДополнительныеОбъекты"`: выгружается только указанная часть
состава (выгрузка зависимостей «по ссылке» при этом продолжает работать). Именно
через него форма выгружает каждый файл отдельным вызовом сервера.

### Ограничения

- Многофайловые режимы работают только при «Выгрузка на сервере» (имя файла —
  путь, доступный серверу). В режиме «на клиенте» команда выдаст сообщение:
  гонять гигабайты через временное хранилище — это второй потолок памяти.
- Регистр без полей отбора (без регистратора и измерений) читается платформой
  одним набором записей и на порции не делится — обработка предупредит об этом
  сообщением.
- При выгрузке по метаданным счетчик «Выгружено объектов» выводится по каждому
  файлу отдельно.
